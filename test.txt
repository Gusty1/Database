/code-review 請分析整個專案架構，幫我拆模組並檢查問題，重點：
1. 代碼結構和模組分層是否合理
2. 潛在 bug 和壞味道
3. 相依性管理是否正確
4. UI component 設計是否合理
5. 有沒有過時或需要更新的依賴
請將分析結果以 Markdown 格式輸出到當前目錄下，檔名為 project-analysis.md

/refactor 根據 project-analysis.md 中檢查結果，優化各模組代碼，保持原功能不變

1. /plan (計畫)
2. /tdd (產出「測試 + 程式架構」)
4. /code-review (程式檢查)
5. /refactor（需要才用）
6. /debug（壞掉才用）
